Sue Jo's Final Project: Recall 8!

CONCEPT

This project demonstrates how physical computing can combine hardware input, software logic, and visual feedback to create an interactive game experience. Inspired by "Say The word on Beat" clips on social media (Link to example video), I designed a web game called Recall8. In the game, players first watch an 8 cards sequence that appears one by one on the screen. After the cards flip, they must repeat the pattern using a physical keypad controller.

SCHEMATIC

For this assignment, I used a 4x4 keypad as the input device, and one RGB LED (common cathode) as the output device. For each LED color, I used 220Ω resistors to limit the current.

Here's my calculation:
V_R = V_pin - V_LED = 5V - 2V = 3V
I = V_R / R = 3V / 0.02A = 150Ω

And then I rounded it up to 220Ω resistor which was provided in the kit.

CIRCUIT

I built a prototype that connects the Arduino, keypad, and RGB LED to a web based game interface. There are three rounds total, and the difficulty increases each round. The sequence is randomly generated each time, so the game will be different every time the user plays it. If they enter the wrong answer, the round restarts with a new sequenece.

  • 1st Round: only numbers
  • 2nd Round: numbers + special characters (* and #)
  • 3rd Round: numbers + special characters + letters (A-D)

An RGB LED provides visual feedback to indicate different stages of the game:
🔵 Playback ⚪️ Start 🟢 Success 🔴 Fail

Together, The final system integrates three components:
Physical keypad controller, web based game interface and the RGB LED ffedback system.

PROGRESS

The screen displays game instructions, the current round number, and prompts that guide the player during the game. The game interface was implemented using HTML, CSS, and JavaScript with p5.js and the WebSerial library.
The web application is responsible for:

  • generating random symbol sequences
  • displaying the sequence visually
  • card flipping animations
  • comparing player input with the correct sequence
  • communicating with the Arduino through serial connection
  • starting and resetting the game

ARDUINO

The Arduino sends the pressed key values to the computer through serial communication, and when the Arduino receives commands from the browser through the serial port, it changes the RGB color accordingly.
The keypad code part was adapted from my Assignment 5 webpage. 


#include <Keypad.h>

// define the number of rows and cols in the keypad
const byte ROWS = 4;
const byte COLS = 4;

// define the key layout corresponding to the keypad buttons
char keys[ROWS][COLS] = {
  {'1','2','3','A'},
  {'4','5','6','B'},
  {'7','8','9','C'},
  {'*','0','#','D'}
};

// define which arduino pins are connected to the keypad row pins and column pins
byte rowPins[ROWS] = {9,8,7,6}; // Arduino pins connected to Row1-Row4
byte colPins[COLS] = {5,4,3,2}; // Arduino pins connected to Col1-Col4

// Initialize an instance of the Keypad class
Keypad keypad = Keypad(makeKeymap(keys), rowPins, colPins, ROWS, COLS);

// define the RGB LED pins
const int redPin = 10;
const int greenPin = 11;
const int bluePin = 12;

void setup() {
  Serial.begin(9600); // start serial communication
  // set the RGB LED pins as outputs
  pinMode(redPin, OUTPUT);
  pinMode(greenPin, OUTPUT);
  pinMode(bluePin, OUTPUT);
  setLED('O'); // OFF
}

void loop() {
 char key = keypad.getKey(); // read the currently pressed key from the keypad
 if(key){ // check if a key is pressed
  // print the value of the pressed key
  Serial.println(key);
 }
 if(Serial.available()){ // check if there is incoming data from serial
  char state = Serial.read(); // read the incoming character from serial
  if(state != '\n' &&  state != '\r'){ // ignore newline and carriage return characters
    setLED(state); // change the LED color based on the received signals
  }
 }
}

// function that controls the RGB LED color based on the signals
void setLED(char state){
  // turn everything off first
  digitalWrite(redPin, LOW);
  digitalWrite(greenPin, LOW);
  digitalWrite(bluePin, LOW);

  if (state == 'O'){ // if state is 'O' (OFF), all colors remain low
  } else if (state == 'W'){ // if state is 'W'(Wait, which means the game is showing off the cards)
    digitalWrite(bluePin, HIGH); // turn on Blue
  } else if (state == 'P'){ // if state is 'P' (Play), turn on White
    digitalWrite(bluePin, HIGH);
    digitalWrite(redPin, HIGH);
    digitalWrite(greenPin, HIGH);
  } else if (state == 'S'){ // if state is 'S' (Success), turn on Green
    digitalWrite(greenPin, HIGH);
  } else if (state == 'F'){ // if state is 'F' (Fail), turn on Red
    digitalWrite(redPin, HIGH);
  }
}
